# ADR-001 (core): 和了判定の内部表現に枚数ベクトル Counts を採用する

- ステータス: Accepted
- 日付: 2026-06-22
- スコープ: core リポ内部（横断ではない）

## 決定

和了形判定・待ち列挙・向聴計算は、長さ34の枚数配列 `Counts`（各種0..4）を内部表現とする。物理牌 `Tile[]`（赤ドラ・id付き）は表示・副露・ドラ集計のために別途保持し、判定直前に `Counts` へ変換する。

## 背景・根拠

和了形判定は「34種の枚数」だけで決まり、個々の牌のidや赤属性は形の成否に無関係。枚数ベクトルなら再帰分解（順子/刻子の剥がし）が添字操作で書け、高速かつバグが少ない。物理牌リストのまま分解すると同種牌の重複処理が煩雑になる。

## 帰結

- 判定ロジックは `Counts` に対する純粋関数（`win.ts`）。テストは `parseHand("123m...")` で `Counts` を直接組める。
- 赤ドラは `kind` を変えない（5として数える）ので、形判定に影響しない。ドラ集計時のみ `Tile.red` を見る。
- これは core 内部の表現選択であり、公開APIの契約（`architecture.md`）には漏らさない。
